# Core Codebase Implementation Log

**Objective:** Upgrading the pure 2D Euclidean CVRP / TSP codebase into an Online Real-World Routing Engine.

## [Phase 1] Real-World Time Matrices (OSRM API Integration)
**Status: COMPLETED (Testing Verified on tsp10, vrp10, vrp20)**

We successfully broke the simulation's reliance on Euclidean math and attached it to real-world Open Source Routing Machine APIs. Below is the **exhaustive tracking list** of precisely what changed mathematically inside the neural network:

### 1. Data Generation (`osrm_data_generator.py`)
* **The Change:** Created a completely new file at the root. 
* **The Logic:** Scrapped the `[0.0, 1.0]` uniform grid point spawner. We now generate GPS coordinate bounds (Latitude/Longitude), query the public OSRM mapping server, and download the exact literal driving times (`[n_nodes x n_nodes]` Time Cost array) between every point.

### 2. Environment Arrays (`TSP/tsp_utils.py` & `VRP/vrp_utils.py`)
* **The Change:** Complete overwrite of the loaders and TensorFlow `Env` state.
* **The Logic:** We intercepted the `DataGenerator` so it loads the new OSRM JSON datasets. Inside the `Env` class, we added a massive new TensorFlow placeholder: `self.time_matrix = tf.placeholder(shape=[None, n_nodes, n_nodes])`.
* **Bug Fixes:** Casted `tf.shape` to `tf.int64` deep in the VRP array masking function to fix a TensorFlow 1.x strict validation clash against `batched_idx`.

### 3. The Reward Calculation (`reward_func` inside Utils)
* **The Change:** Annihilated the original Euclidean calculator.
* **The Exact Math:** Previously, the `reward_func` accepted `sample_solution` (an array of `[x,y]` coordinates) and calculated `tf.sqrt(dx^2 + dy^2)`. 
* **The New Math:** It now takes `idxs` (the strict node sequence generated by the Actor) and directly uses `tf.gather_nd` to act as a 3D index-lookup table against `env.time_matrix`. The entire reward is now the literal sum of the driving seconds returned by OSRM.
* **The Unserved Penalty Fix:** Added a massive mathematical penalty (`unserved_demand * 10000.0`) to the reward function. This closed a critical RL loophole where the agent could achieve a "perfect" score of `0s` by infinitely looping at the Depot. The agent is now strictly forced to prioritize visiting all customers.

### 4. Attention Mechanism, Actor, and Critic (`attention_agent.py`)
* **The Change:** Rewired the TensorFlow Feed-Dictionaries and the `R` (Reward) execution parameter.
* **The Feed Dict:** In `run_train_step` and `evaluate`, the `feed_dict` now unpacks the `data` array perfectly into the `self.env.time_matrix` placeholder alongside the `demand` and `input_pnt`.

**Crucial Neural Network Architecture Note:**
In Phase 1, **LinearEmbedding**, the **Attention Mechanism**, and the **Encoder/Decoder** structures remain entirely unmodified. 
* The Actor still embeds the 2D `[x,y]` coordinates linearly as spatial anchors.
* *How it Learns without Graph Embeddings:* Even though the Actor only "sees" a 2D coordinate, the **Critic Loss** and **Actor Policy Gradient** are now completely driven by the `R` (driving seconds) value outputted by our new lookup. Therefore, the Attention Mechanism organically learns to prioritize nodes that minimize time (ignoring literal Euclidean distance), simply because the backpropagation forces it to! 
*(Note: Transitioning the `LinearEmbedding` into a true Adjacency Matrix `Graph Neural Network` is officially scheduled for **Phase 4**).*
